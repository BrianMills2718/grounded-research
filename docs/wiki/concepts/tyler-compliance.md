# Tyler Compliance

Tyler compliance means the live runtime and exported artifacts match the Tyler
V1 requirements tracked in the evidence ledger. It is not proven by an old plan
being marked complete, and it is not reopened by an archived plan title.

## Maintainer Rule

Use `docs/TYLER_SPEC_GAP_LEDGER.md` for requirement-level truth and
`docs/TYLER_EXECUTION_STATUS.md` for current status. Use historical plans only
as provenance for why a row was closed, downgraded, or left on watch.

## Current Boundary

Documentation cleanup may improve discoverability, gates, and orientation. It
must not change Tyler Stage 1-6 behavior unless a new plan names the exact Tyler
ledger row, acceptance criteria, and verification evidence.
