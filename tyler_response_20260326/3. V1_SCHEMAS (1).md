# Evidence-Grounded Reasoning Orchestrator — V1 Data Contracts

> Pydantic model definitions for every stage's input and output.
> These are executable contracts — enforced at the API level via structured output.
> Distilled from `V0/SCHEMAS.md` using V1 pruning decisions.
>
> **How to read this:** Each schema is a Python class using Pydantic. Fields have types and descriptions.
> The developer's AI coding tool converts these directly into working code.
> Pydantic auto-generates JSON Schema for Claude (json_schema) and Gemini (responseJsonSchema).

## Schema Categories

**Model-output schemas** — Enforced via structured output. Must be simple enough for reliable generation.
- Stage 1: DecompositionResult (Gemini 3.1 Pro)
- Stage 3: AnalysisObject (GPT-5.4, Claude Opus 4.6, Gemini 3.1 Pro — each independently)
- Stage 4: ClaimExtractionResult (GPT-5.4) — may require multiple calls
- Stage 5: ArbitrationAssessment (Claude Opus 4.6) — one per dispute
- Stage 6b: SynthesisReport (TBD model)
- StageSummary (whichever model runs each stage)

**Orchestrator-constructed schemas** — Built by Python code, not model output. Can be arbitrarily complex.
- Stage 2: EvidencePackage (assembled from Tavily/Exa API responses)
- Stage 5: VerificationResult (combines search results + model arbitration)
- PipelineState (wrapper carrying all data between stages)
- TraceFile (concatenation of all artifacts)

*V1 removals: Stage 1v (DecompositionValidation), Stage 6a steering schemas (SteeringQuestion, UserSteeringInterrupt, SteeringResponse, UserSteeringResponse). See V1_DESIGN.md V1 Limitations section.*

```python
from enum import Enum
from pydantic import BaseModel, Field
from typing import Optional


# --- Enums ---

class EvidenceLabel(str, Enum):
    """Hierarchy from DESIGN.md constraint #5. Higher = more trustworthy."""
    VENDOR_DOCUMENTED = "vendor_documented"          # 1.0 — official docs, specs
    EMPIRICALLY_OBSERVED = "empirically_observed"    # 0.8 — benchmarks, practitioner reports
    MODEL_SELF_CHARACTERIZATION = "model_self_characterization"  # 0.5 — model claims about itself
    SPECULATIVE_INFERENCE = "speculative_inference"  # 0.3 — reasoning without direct evidence

class DisputeType(str, Enum):
    """Five-type taxonomy from the disagreement classification framework."""
    EMPIRICAL = "empirical"                # Facts/sources — resolve with more evidence
    INTERPRETIVE = "interpretive"          # What evidence means — reasoning critique
    PREFERENCE_WEIGHTED = "preference_weighted"  # Optimization criteria — user context needed
    SPEC_AMBIGUITY = "spec_ambiguity"      # Question not constrained enough — user input needed
    OTHER = "other"                        # V1 addition (Task #31) — edge case catch-all, uses existing description field for routing context

class ClaimStatus(str, Enum):
    """Lifecycle: initial → {supported|contested|contradicted|insufficient} → {verified|refuted|unresolved}"""
    INITIAL = "initial"                    # Extracted, not yet cross-referenced
    SUPPORTED = "supported"                # Multiple models agree, evidence aligns
    CONTESTED = "contested"                # Models disagree, needs resolution
    CONTRADICTED = "contradicted"          # Evidence directly conflicts
    INSUFFICIENT_EVIDENCE = "insufficient_evidence"  # Can't determine
    VERIFIED = "verified"                  # Post-Stage 5: confirmed by targeted evidence
    REFUTED = "refuted"                    # Post-Stage 5: disproven by targeted evidence
    UNRESOLVED = "unresolved"              # Post-Stage 5: still can't determine

class DisputeStatus(str, Enum):
    UNRESOLVED = "unresolved"
    RESOLVED = "resolved"
    DEFERRED_TO_USER = "deferred_to_user"  # Preference/ambiguity/other — surfaced to user via terminal interrupt
    LOGGED_ONLY = "logged_only"            # Not decision-critical, recorded but not pursued

class ConfidenceLevel(str, Enum):
    """Used for confidence assessment. Telemetry signal only — never used for arbitration decisions."""
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

class ResolutionOutcome(str, Enum):
    CLAIM_SUPPORTED = "claim_supported"
    CLAIM_REFUTED = "claim_refuted"
    EVIDENCE_INSUFFICIENT = "evidence_insufficient"
    INTERPRETATION_REVISED = "interpretation_revised"
```

## Stage 1 — Intake & Decomposition

**Model:** Gemini 3.1 Pro | **Type:** Model-output schema

```python
class SubQuestion(BaseModel):
    id: str = Field(description="Format: Q-{n}")
    question: str = Field(description="Specific, researchable question")
    type: str = Field(description="'empirical' | 'interpretive' | 'preference'")
    research_priority: str = Field(description="'high' | 'medium' | 'low'")
    search_guidance: str = Field(description="What to look for — source types, keywords, practitioner signals")

class ResearchPlan(BaseModel):
    what_to_verify: list[str] = Field(description="Key factual claims to check")
    critical_source_types: list[str] = Field(description="Types of sources that matter most, e.g., 'official docs', 'practitioner benchmarks'")
    falsification_targets: list[str] = Field(description="What evidence would change the answer. N/A valid for non-falsifiable topics.")

class DecompositionResult(BaseModel):
    """Stage 1 output. Breaks the user query into researchable components."""
    core_question: str = Field(description="The central question or decision, restated precisely")
    sub_questions: list[SubQuestion] = Field(description="2-6 sub-questions to research")
    optimization_axes: list[str] = Field(description="Likely tradeoff dimensions, e.g., 'cost vs. quality', 'simplicity vs. extensibility'")
    research_plan: ResearchPlan
    stage_summary: StageSummary
```

**Validation rules:**
- `sub_questions` must have 2-6 entries
- Each sub-question `id` must be unique (Q-1, Q-2, ...)
- At least one sub-question must be `research_priority: "high"`
- `falsification_targets` must not be empty (may contain "N/A" for non-falsifiable topics)

*V1 removals: AmbiguousTerm class (Task #3: CUT — sub-questions disambiguate implicitly), `ambiguous_terms` field, `complexity_assessment` field, ComplexityLevel enum (Task #6: CUT — fixed budgets).*

## Stage 2 — Broad Retrieval & Evidence Normalization

**Tools:** Tavily + Exa | **Type:** Orchestrator-constructed

```python
class Finding(BaseModel):
    """A specific piece of information extracted from a source."""
    finding: str = Field(description="Factual statement extracted from the source")
    evidence_label: EvidenceLabel
    original_quote: Optional[str] = Field(default=None, description="Preserved verbatim for conflict-bearing findings — paraphrasing introduces harmonization bias")

class Source(BaseModel):
    id: str = Field(description="Format: S-{n}, globally unique across the pipeline")
    url: str
    title: str
    source_type: str = Field(description="'official_docs' | 'academic' | 'practitioner_report' | 'forum' | 'blog' | 'news'")
    quality_score: float = Field(ge=0.0, le=1.0, description="FINAL blended score: authority (URL lookup table) + freshness decay + staleness adjustments. See Build Plan Stage 2 Steps 7-9 for the scoring pipeline. Unknown sources default to authority score 0.5.")
    publication_date: Optional[str] = Field(default=None, description="ISO 8601 date of original publication. From Exa publishedDate, Tavily published_date (with topic='news'), or extracted from content. Used for freshness decay scoring. If unavailable, freshness adjustment is skipped and authority score is used directly.")
    retrieval_date: str = Field(description="ISO 8601 date")
    key_findings: list[Finding]

class SubQuestionEvidence(BaseModel):
    sub_question_id: str = Field(description="References DecompositionResult.sub_questions[].id")
    sources: list[Source]
    meets_sufficiency: bool = Field(description="True if min 2 independent high-quality sources found")
    gap_description: Optional[str] = Field(default=None, description="What's missing, if below sufficiency threshold")

class EvidencePackage(BaseModel):
    """Stage 2 output. Assembled by orchestrator from search API responses.
    THIS IS QUALITY BOTTLENECK #1 — shallow sources here = polished noise downstream."""
    sub_question_evidence: list[SubQuestionEvidence]
    total_queries_used: int
    queries_per_sub_question: dict[str, int] = Field(description="Map of Q-{n} → query count")
    stage_summary: StageSummary
```

**Validation rules:**
- Every `sub_question_id` must reference an ID from Stage 1's `sub_questions`
- Source `id` values must be globally unique (no collisions across sub-questions)
- `quality_score` is the FINAL blended output of the scoring pipeline (authority + freshness + staleness). It must be in range [0.1, 1.0] — the 0.1 floor comes from the deprecation keyword penalty. Authority floor of 0.4 applies to seminal sources (RFCs, specs). See Build Plan Stage 2 Steps 7-9.
- Budget enforcement: max 4 queries per sub-question (hard cap, named constant)

*V1 simplification: early-stop heuristic ("stop when last 2 queries yield <10% novel info") removed — hard cap only.*

*V1 removals: EchoDetection class and `echo_detection` field on Source (Task #17: DEFER), `grok_trend_seeds` field (Task #14: DEFER). Note: Tavily's `score` field is relevance, not credibility — do not use as quality_score.*

## Stage 3 — Independent Candidate Generation

**Models:** GPT-5.4, Claude Opus 4.6, Gemini 3.1 Pro (parallel) | **Type:** Model-output schema

```python
class Claim(BaseModel):
    id: str = Field(description="Format: C-{n}, unique within this analysis")
    statement: str = Field(description="Specific, falsifiable claim")
    evidence_label: EvidenceLabel
    source_references: list[str] = Field(description="List of S-{n} IDs from the evidence package")
    confidence: ConfidenceLevel = Field(description="TELEMETRY ONLY — not used for arbitration")

class Assumption(BaseModel):
    id: str = Field(description="Format: A-{n}, unique within this analysis")
    statement: str = Field(description="Stated explicitly — first-class citizen, not afterthought")
    depends_on_claims: list[str] = Field(description="List of C-{n} IDs this assumption supports")
    if_wrong_impact: str = Field(description="What changes if this assumption is false")

class CounterArgument(BaseModel):
    """Required field — models MUST argue against their own recommendation.
    Design constraint #9: anti-conformity as protocol rule."""
    argument: str = Field(description="Strongest case against the model's own recommendation")
    strongest_evidence_against: str = Field(description="Most compelling counter-evidence")
    counter_confidence: ConfidenceLevel = Field(description="TELEMETRY ONLY")

class AnalysisObject(BaseModel):
    """Stage 3 output. One per model. Models are anonymized as Analyst A/B/C.
    Each model receives a DIFFERENT reasoning frame (assigned by orchestrator).
    No model sees another's output — independence is enforced by architecture."""
    model_alias: str = Field(description="'A', 'B', or 'C' — anonymized")
    reasoning_frame: str = Field(description="Assigned frame: 'step_back_abstraction' | 'structured_decomposition' | 'verification_first'")
    recommendation: str = Field(description="Bottom-line answer — a recommendation, not a survey")
    claims: list[Claim] = Field(description="All claims supporting the recommendation")
    assumptions: list[Assumption] = Field(description="All assumptions, stated explicitly")
    evidence_used: list[str] = Field(description="S-{n} IDs of sources actually referenced")
    counter_argument: CounterArgument
    falsification_conditions: list[str] = Field(description="What would prove this recommendation wrong")
    reasoning: str = Field(description="How the model got from evidence to conclusion")
    stage_summary: StageSummary
```

**Validation rules:**
- `claims` must have at least 3 entries
- Every `source_references` entry must reference a valid S-{n} from Stage 2
- `evidence_used` must be a subset of available source IDs
- `counter_argument` must not be empty/generic — prompt asks for specific claims or evidence references (V1: prompt-enforced only, no automated validation — see note below)
- `falsification_conditions` must have at least 1 entry
- All claim IDs unique within this analysis (C-1, C-2, ...). Note: claim IDs are model-local. Stage 4 re-IDs them globally.

*V1 change: Counter-argument specificity check downgraded from Tier 4 validation to Tier 5 (prompt instruction only, no automated enforcement). Prompt still asks for specific references but no reject-and-retry. ~70-80% effective with frontier models.*

## Stage 4 — Claim Extraction & Dispute Localization

**Model:** GPT-5.4 | **Type:** Model-output schema

> **Implementation note:** This is the most complex model output. For complex queries with many claims,
> the orchestrator may split this into multiple API calls (e.g., extract claims per model, then cross-reference).
> The schema defines the final artifact shape regardless of how many calls produce it.

```python
class ClaimLedgerEntry(BaseModel):
    """Atomic unit of the pipeline's analytical state. Every downstream decision traces back here."""
    id: str = Field(description="Format: C-{n}, globally unique across all models")
    statement: str = Field(description="Canonical statement — disambiguated and deduplicated")
    source_models: list[str] = Field(description="Which model aliases made this claim: ['A'], ['A','B'], etc.")
    evidence_label: EvidenceLabel
    source_references: list[str] = Field(description="S-{n} IDs from evidence package")
    status: ClaimStatus
    supporting_models: list[str] = Field(description="Model aliases that agree with this claim")
    contesting_models: list[str] = Field(description="Model aliases that disagree")
    related_assumptions: list[str] = Field(description="A-{n} IDs of assumptions this claim depends on")

class AssumptionSetEntry(BaseModel):
    """First-class citizen — not an afterthought. Subject to same evidence mapping as claims."""
    id: str = Field(description="Format: A-{n}, globally unique")
    statement: str
    source_models: list[str] = Field(description="Which model aliases made this assumption")
    dependent_claims: list[str] = Field(description="C-{n} IDs of claims that depend on this assumption")
    if_wrong_impact: str
    shared_across_models: bool = Field(description="True if 2+ models share this assumption")

class ModelPosition(BaseModel):
    model_alias: str
    position: str = Field(description="What this model believes about the disputed claims")

<!-- ENFORCEMENT: resolution_routing and ExtractionStatistics are ORCHESTRATOR-COMPUTED (Tier 2).
     See DECISIONS.md enforcement framework (2026-03-19). The model does not own these fields.
     The orchestrator computes resolution_routing from type + decision_critical.
     The orchestrator computes ExtractionStatistics from actual array lengths. -->

class DisputeQueueEntry(BaseModel):
    """Derived work queue — materialized from the claim graph. Not a parallel data store."""
    id: str = Field(description="Format: D-{n}")
    type: DisputeType
    description: str = Field(description="Plain language: what's being disputed and why it matters")
    claims_involved: list[str] = Field(description="C-{n} IDs of the conflicting claims")
    model_positions: list[ModelPosition]
    decision_critical: bool = Field(description="v1 proxy: removing this dispute's claims changes which alternative leads")
    decision_critical_rationale: str = Field(description="Why this does/doesn't affect the recommendation")
    status: DisputeStatus
    resolution_routing: str = Field(description="ORCHESTRATOR-COMPUTED — do not rely on model output. Computed from type + decision_critical. See TECH_SPEC.md enforcement framework.")

class ExtractionStatistics(BaseModel):
    """ORCHESTRATOR-COMPUTED (Tier 2). Telemetry for observability — not used in pipeline logic.
    Orchestrator computes all counts from actual array lengths after model output. See DECISIONS.md enforcement framework."""
    total_claims: int
    total_assumptions: int
    total_disputes: int
    disputes_by_type: dict[str, int] = Field(description="DisputeType → count")
    decision_critical_disputes: int
    claims_per_model: dict[str, int] = Field(description="Model alias → claim count")

class ClaimExtractionResult(BaseModel):
    """Stage 4 output. THIS IS QUALITY BOTTLENECK #2.
    These three artifacts are the canonical state for everything downstream.
    Synthesis operates on these, not on raw model outputs."""
    claim_ledger: list[ClaimLedgerEntry]
    assumption_set: list[AssumptionSetEntry]
    dispute_queue: list[DisputeQueueEntry]
    statistics: ExtractionStatistics
    stage_summary: StageSummary
```

**Validation rules:**
- All IDs globally unique (no C-{n} collisions across the full ledger)
- Every `claims_involved` in dispute_queue must reference valid claim IDs in the ledger
- Every `dependent_claims` in assumption_set must reference valid claim IDs
- Every `related_assumptions` in claim_ledger must reference valid assumption IDs
- Referential integrity: no dangling references in any direction
- `decision_critical` disputes where `type` is `empirical` or `interpretive` must route to `stage_5_evidence` or `stage_5_arbitration`
- `decision_critical` disputes where `type` is `preference_weighted`, `spec_ambiguity`, or `other` must route to `stage_6_user_input`
- Non-decision-critical disputes must route to `logged_only`
- `statistics` counts must match actual array lengths (NOTE: orchestrator-computed — see enforcement framework)

*V1 removal: Evidence-label leakage check (Task #30: DEFER). Nice v2 guardrail — for V1, trust the model's labeling. Evidence trail makes any mislabeling visible in the trace file.*

### Model output (one per investigated dispute):

```python
class ArbitrationAssessment(BaseModel):

    Design constraint #2: critique targets claims, not responses."""
    dispute_id: str = Field(description="D-{n} from the dispute queue")
    new_evidence_summary: str = Field(description="What the targeted search found")
    reasoning: str = Field(description="Step-by-step assessment of the dispute given new evidence")
    resolution: ResolutionOutcome
    updated_claim_statuses: list["ClaimStatusUpdate"] = Field(description="One entry per claim affected by this dispute")

class ClaimStatusUpdate(BaseModel):
    """Typed update record — replaces generic dict for type safety."""
    claim_id: str = Field(description="C-{n} of the claim being updated")

    confidence_in_resolution: ConfidenceLevel = Field(description="TELEMETRY ONLY")
```

### Full stage output (orchestrator-assembled):

```python
class AdditionalSource(BaseModel):
    """Evidence found during targeted verification. Uses simplified key_findings (strings, not Finding objects)
    because these are quick-pass search results, not the full evidence normalization of Stage 2."""
    source_id: str = Field(description="S-{n}, globally unique — must not collide with Stage 2 IDs")
    url: str
    title: str
    quality_score: float = Field(ge=0.0, le=1.0)
    key_findings: list[str] = Field(description="Simplified findings as strings — not full Finding objects like Stage 2")
    retrieved_for_dispute: str = Field(description="D-{n} this was retrieved to investigate")

class VerificationResult(BaseModel):
    """Stage 5 output. Assembled by orchestrator from search results + model arbitration.
    Only fires on unresolved empirical/interpretive disputes that are decision-critical."""
    disputes_investigated: list[ArbitrationAssessment]
    additional_sources: list[AdditionalSource]
    updated_claim_ledger: list[ClaimLedgerEntry] = Field(description="Full ledger with updated statuses")
    updated_dispute_queue: list[DisputeQueueEntry] = Field(description="Full queue with updated statuses")
    search_budget: dict[str, int] = Field(description="D-{n} → queries used")
    rounds_used: int = Field(description="1 or 2 (max 2 per design)")
    stage_summary: StageSummary
```

**Validation rules:**
- Every `dispute_id` in ArbitrationAssessment must reference a dispute from Stage 4's queue
- Only disputes with `type` in {empirical, interpretive} AND `decision_critical == true` should appear
- `additional_sources` IDs must not collide with Stage 2 source IDs
- Updated claim statuses must be from the post-verification set: {verified, refuted, unresolved}
- Budget enforcement: max 3 queries per dispute, max 2 rounds total

```python
class Tradeoff(BaseModel):

    summary: str
    resolution: str = Field(description="How it was resolved or why it remains open")

    statement: str
    final_status: ClaimStatus
    resolution_path: str = Field(description="How this claim was validated/invalidated")

class EvidenceTrailEntry(BaseModel):
    source_id: str
    url: str
    quality_score: float
    key_contribution: str = Field(description="What this source contributed to the analysis")
    conflicts_resolved: Optional[list[str]] = Field(default=None, description="D-{n} IDs if this source helped resolve disputes")

class SynthesisReport(BaseModel):
    """Stage 6b output. The final deliverable. 11 components in 3 tiers.
    V1 runtime checks: grounding (cites claims) + zombie (no refuted alternatives).
    See EVAL_FLAG EF-10 for remaining anti-patterns evaluated in Phase 5."""

    # Tier A: Executive
    executive_recommendation: str = Field(description="1-3 paragraphs. Bottom-line answer as a recommendation, not a survey.")
    conditions_of_validity: list[str] = Field(description="What would flip the recommendation. Stated upfront.")
    decision_relevant_tradeoffs: list[Tradeoff]

    # Tier B: Analytical
    disagreement_map: list[DisagreementMapEntry]
    preserved_alternatives: list[PreservedAlternative]
    key_assumptions: list[KeyAssumption]
    confidence_assessment: list[ConfidenceAssessment]
    process_summary: list[StageSummary] = Field(description="Compiled from all stage summaries. Scannable in 30 seconds.")

    # Tier C: Evidentiary
    claim_ledger_excerpt: list[ClaimLedgerExcerpt] = Field(description="Decision-critical claims only")
    evidence_trail: list[EvidenceTrailEntry]
    evidence_gaps: list[str] = Field(description="What the system looked for but couldn't find")

    reasoning: str = Field(description="How the synthesis model arrived at this recommendation from the pipeline state")
    stage_summary: StageSummary
```

**Validation rules:**
- `conditions_of_validity` must have at least 1 entry
- **Zombie-alternative check (V1 runtime — Tier 4):** `preserved_alternatives` must not include alternatives eliminated by critique. Validation: every `supporting_claims` ID must reference a claim whose `status` is not `refuted`.
- Every `supporting_claims` ID must reference a valid claim in the ledger
- `evidence_trail` must include all sources that contributed to the recommendation
- `evidence_gaps` should list what was searched for but not found — not just "none"
- **Post-synthesis grounding check (V1 runtime — Tier 4):** `executive_recommendation` must contain at least one claim reference (C-{n}) from the claim ledger. A recommendation that cites no claims is disconnected from the pipeline's evidence chain. Reject and retry once.
- `executive_recommendation` must not contain more than 2 levels of conditional nesting
- `process_summary` must contain one StageSummary per stage that executed (5 if Stage 5 was skipped, 6 if all stages ran). Stage 3 contributes one combined summary, not one per model — individual model summaries are in the trace file.

## Pipeline State

**Type:** Orchestrator-constructed

```python
class PipelineError(BaseModel):
    stage: int
    error_type: str = Field(description="'api_failure' | 'invalid_json' | 'timeout' | 'validation_error' | 'budget_exceeded'")
    message: str
    recoverable: bool
    action_taken: str = Field(description="What the orchestrator did: 'retried' | 'skipped_model' | 'used_fallback' | 'aborted'")

class PipelineState(BaseModel):
    """The master state object carried through the pipeline.
    Serialized to JSON as the trace file."""
```

## Skip and Branch Conditions

These are deterministic orchestrator logic — not model judgment calls.

### Stage 5 skip condition
```python
# Skip Stage 5 entirely if no disputes need investigation
should_skip_stage_5 = not any(
    d.type in (DisputeType.EMPIRICAL, DisputeType.INTERPRETIVE)
    and d.decision_critical == True
    and d.status == DisputeStatus.UNRESOLVED
    for d in stage_4_result.dispute_queue
)
```

### Stage 6 interrupt trigger
```python
# Fire user-steering interrupt if preference/ambiguity/other disputes remain
# current_dispute_queue = stage_5_result.updated_dispute_queue (if Stage 5 ran)
#                       = stage_4_result.dispute_queue (if Stage 5 was skipped)
interrupt_disputes = [d for d in current_dispute_queue
    if d.type in (DisputeType.PREFERENCE_WEIGHTED, DisputeType.SPEC_AMBIGUITY, DisputeType.OTHER)
    and d.decision_critical == True
    and d.status == DisputeStatus.UNRESOLVED]

should_fire_interrupt = len(interrupt_disputes) > 0
# If >2 qualifying disputes, orchestrator selects the 2 with highest decision-criticality rationale
# V1 change: dispute selection moved from model to orchestrator (consistent with Stage 6a schema removal)
```

### Decision-critical claim set (orchestrator-derived)

Claims do not carry a `decision_critical` field — only disputes do. To identify decision-critical claims for Stage 6b synthesis, the orchestrator derives the set from the dispute queue:

```python
# Claims involved in any decision-critical dispute
decision_critical_claim_ids = set()
for d in current_dispute_queue:
    if d.decision_critical:
        decision_critical_claim_ids.update(d.claims_involved)
```

This derived set is used by the Stage 6b prompt template to split the claim ledger into decision-critical and non-critical subsets. See `V1_PROMPTS.md` Stage 6b USER template.

### Minimum model threshold (Stage 3)
```python
# Pipeline continues with 2 models if one fails. Below 2 = abort.
minimum_stage_3_models = 2
```

## Decision-Criticality: v1 Operationalization

A dispute is **decision-critical** if removing the disputed claims from the claim graph would change which alternative the pipeline would recommend.

**v1 proxy (for GPT-5.4 in Stage 4):**
The model evaluates: "If this dispute were resolved in the opposite direction, would the recommendation change?" If yes → `decision_critical: true`. The `decision_critical_rationale` field must explain the reasoning.

This is imperfect but implementable. A more rigorous approach (actually removing claims and re-running synthesis) is a v2 optimization.

## ID Scheme

| Prefix | Object | Example | Scope |
|--------|--------|---------|-------|
| Q- | Sub-question | Q-1, Q-2 | Unique within Stage 1 |
| S- | Source | S-1, S-2 | Globally unique across pipeline |
| C- | Claim | C-1, C-2 | Model-local in Stage 3, globally re-ID'd in Stage 4 |
| A- | Assumption | A-1, A-2 | Model-local in Stage 3, globally re-ID'd in Stage 4 |
| D- | Dispute | D-1, D-2 | Globally unique (created in Stage 4) |

**Stage 4 re-identification:** When Stage 4 extracts claims from three models, it deduplicates and assigns new global IDs. A claim made by models A and B becomes one ledger entry with `source_models: ["A", "B"]`. The original model-local IDs are not preserved in the canonical ledger.
